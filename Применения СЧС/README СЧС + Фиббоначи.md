# Документация:  
## **СЧС-анализ последовательности Фибоначчи по 4 типам совпадений**  
*Structural Numerical Symmetry (SNS) Analysis of the Fibonacci Sequence — 4-Type Classification*  
**Автор**: Ющенко Михаил Юрьевич  
**Версия**: 1.0  
**Дата**: 31 января 2026 г.


### 1. Назначение

Скрипт предназначен для **применения закона Структурной Числовой Симметрии (СЧС)** к первым `N` членам последовательности Фибоначчи и **классификации каждого числа по строгой 4-типовской системе**, предложенной автором.  

Цель — демонстрация **универсальности СЧС**: любой член последовательности **обязательно** попадает в один из четырёх типов, **исключая возможность «отсутствия структуры»**.


### 2. Основные принципы СЧС

СЧС — фундаментальный закон десятичной системы, проявляющийся при следующей операции:

1. Возьмём натуральное число \( N \).

2. Разобьём его на `m` частей **максимально равной длины** (слева направо).

3. Каждую часть умножим на целый множитель `k`, **сохранив исходную длину части** (дополняя слева нулями при необходимости).

4. Объединим части — получим строку **PQ**.

5. Вычислим NK = N*k

6. Сравним **PQ** и **NK** по следующей **четырёхтиповой классификации**.


### 3. Типология совпадений (обязательная)

| Тип | Условие | Обозначение в коде |
|-----|--------|-------------------|
| **1** | Полное совпадение: `PQ == NK` | `"1_full"` |
| **2** | Совпадают **и начало (≥2 цифр), и конец (≥2 цифр)** | `"2_both_ends"` |
| **3** | Совпадает **только конец (≥1 цифра)**, но не всё число и не начало | `"3_suffix_only"` |
| **4** | **Нет совпадений** (ни начала, ни конца) | `"4_none"` |

> **Примечание**: Согласно моей теории, **тип 4 не должен встречаться** в натуральных последовательностях — что подтверждается эмпирически.


### 4. Входные параметры

| Параметр | По умолчанию | Описание |
|--------|--------------|--------|
| `N_TERMS` | `200` | Количество членов Фибоначчи для анализа (начиная с \( F_1 = 1 \)) |
| `M_SNS` | `2` | Число частей `m` при разбиении числа |
| `K_SNS` | `3` | Множитель `k` для умножения частей |
| `VIDEO_FPS` | `10` | Частота кадров анимации (только для визуализации) |
| `PLOT_SIZE` | `(1280, 720)` | Разрешение графиков |


### 5. Выходные файлы

Скрипт генерирует **четыре файла** в рабочей директории:

| Файл | Формат | Содержание |
|------|--------|-----------|
| `fibonacci_sns_4type_statistics.txt` | TXT | Сводная статистика по 4 типам, параметры анализа, метаданные автора |
| `fibonacci_sns_4type_data.csv` | CSV | Таблица: индекс, значение \( F_n \), тип СЧС, длина совпадающего суффикса |
| `fibonacci_sns_4type_plot.png` | PNG | Статический график: номер числа vs длина суффикса, цвет — по типу совпадения |
| `fibonacci_sns_4type_animation.mp4` | MP4 | Анимация эволюции СЧС по мере роста \( n \) (требуется `ffmpeg`) |

> **Цветовая схема графиков**:  
> - Тип 1 (полное) → **зелёный**  
> - Тип 2 (начало+конец) → **синий**  
> - Тип 3 (только конец) → **оранжевый**  
> - Тип 4 (нет) → **светло-серый**


### 6. Оптимизации

Для повышения производительности при работе с большими числами применены:

- `@inbounds` — отключение проверок границ в циклах сравнения.
- `sizehint!` — предварительное выделение памяти для массивов.
- Прямое посимвольное сравнение строк **без создания подстрок**.
- Устранение избыточного выравнивания `lpad` при сравнении `PQ` и `NK` (сохраняется оригинальная структура).

> **Важно**: СЧС корректно работает **только с исходными строками** — ведущие нули в частях сохраняются, но при сравнении **не дополняются** до общей длины, так как это искажает структурную природу совпадения.

### 9. Научное значение


Результат подтверждает **мою теорию** о том, что **всякий натуральный объект подчиняется СЧС**, и **отсутствие совпадений (тип 4) невозможно в регулярных числовых последовательностях**.  
Последовательность Фибоначчи, несмотря на свой иррациональный предел (золотое сечение), **полностью укладывается в структурную сетку СЧС** — что указывает на **глубинную дискретную природу роста в природе**.
